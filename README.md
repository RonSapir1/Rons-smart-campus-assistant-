# 🎓 Smart Campus Assistant (AI-Powered RAG System) by Ron Sapir

מערכת בינה מלאכותית חכמה לקמפוס, המספקת לסטודנטים מענה מיידי, מדויק ואמין לשאלות בנושאי הקמפוס. המערכת מבוססת על ארכיטקטורת **RAG (Retrieval-Augmented Generation)** להבטחת אפס הזיות , ארוזה בקונטיינרים.

---

## 🏗️ ארכיטקטורת המערכת (System Architecture)

המערכת בנויה בארכיטקטורת Microservices מודרנית וקלת-משקל:

1. **Frontend (ממשק משתמש - צד לקוח):** * ווידג'ט צ'אט מרחף, ריאקטיבי ומעוצב, המוטמע על גבי "אתר הקמפוס".
   * נכתב ב-Vanilla HTML/CSS/JS לקבלת ביצועים מקסימליים ללא תלויות חיצוניות.
   * מוגש על ידי שרת **Nginx** קל ומהיר.
2. **Backend (שרת ולוגיקה):**
   * שרת מבוסס Python ו **FastAPI**.
   * משמש כ"שומר סף" (Middleware) הכולל ניהול CORS מאובטח ותקשורת אסינכרונית מול ה-AI.
3. **Data & AI (נתונים ומנוע שפה):**
   * **מסד נתונים מקומי (SQLite):** מאחסן את נתוני האמת של הקמפוס. הנתונים מנוהלים ונשמרים בצורה בטוחה באמצעות Docker Volumes (כך שעדכון במסד הנתונים משתקף מיד במוח של ה-AI).
   * **מנוע AI (Gemini 2.5 Flash):** מופעל בתצורת `JSON Mode` נוקשה, המחייבת החזרת אובייקט הכולל: תשובה, קטגוריה, וציון ביטחון (Confidence Score).
   * **מנגנון Fallback:** השרת חוסם אוטומטית תשובות עם ציון ביטחון נמוך כדי למנוע הטעיית סטודנטים.

🚀 הוראות התקנה והרצה (Docker Deployment)
המערכת תוכננה ל-Zero Setup. אין צורך בהתקנת סביבות פייתון מקומיות.
שלב 1 - פתחו את הטרמינל בתיקיית השורש של הפרויקט (היכן שנמצא docker-compose.yml).
שלב 2 - הריצו את פקודת הבנייה וההפעלה: docker compose up --build .
שלב 3 - הגישה למערכת: ממשק המשתמש (הצ'אטבוט והאתר): פתחו דפדפן ונווטו לכתובת http://localhost
שרת ה-API הפנימי: מאזין ברקע בכתובת http://localhost:8000
לעצירת המערכת: לחצו Ctrl + C בטרמינל, או הריצו docker compose down.

🛠️ פתרון תקלות נפוצות (Troubleshooting)
במהלך פיתוח ופריסת המערכת בנינו מנגנונים להתמודדות עם אתגרי Production:

שגיאת uvicorn: executable file not found במהלך בניית הדוקר:
בעיה זו נובעת מ-Cache ישן של דוקר שלא קלט עדכונים בקובץ ה-requirements.txt.
פתרון: הריצו בנייה נקייה ללא מטמון: docker compose build --no-cache ולאחר מכן docker compose up.

הצ'אט עולה אך הבוט לא עונה (שגיאת CORS / Network):
אם ה-Frontend לא מצליח לדבר עם ה-Backend, ודאו שאתם גולשים לכתובת http://localhost בדיוק, ולא ל-127.0.0.1 או לקובץ ה-HTML המקומי, כדי שחוקי ה-CORS שהגדרנו ב-FastAPI יאפשרו את התקשורת.

ה-AI מפסיק להגיב או מחזיר שגיאת 429 (Rate Limit):
אם נשלחו יותר מדי בקשות בזמן קצר, ייתכן שמיציתם את מכסת הבקשות (Quota) של המפתח החינמי.
פתרון: הנפיקו מפתח חדש ב-Google AI Studio, עדכנו את קובץ ה-.env בתיקיית ה-backend, והפעילו מחדש את הקונטיינר. (כחלק מתוכנית ההמשך, יתווסף Rate Limiter בשרת ה-FastAPI למניעת התקפות ספאם).

הבוט עונה "אני לא בטוח" (Fallback) על נתונים שאמורים להיות קיימים:
ודאו שקובץ ה-campus.db מוזן בנתונים ושמורות בו הרשומות הנכונות. ניתן להריץ את סקריפט ה-seed.py כדי לאפס ולמלא את מסד הנתונים מחדש.
